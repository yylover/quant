# 完整交易系统流程说明

本说明对应策略文件：`strategies/complete_flow_system.py`，基于「最完善的交易流程图」实现可运行、可回测的 10 步流程。

---

## 流程与代码对应关系

| 步骤 | 流程图名称 | 实现要点 |
|------|------------|----------|
| **01** | 辨趋势 | 双均线（短/长）：短 > 长 → 上涨；短 < 长 → 下跌；均线缠绕 → 震荡 |
| **02** | 判方向 | 趋势为上涨 → 主多；下跌 → 主空（本策略仅做多）；震荡 → 偏多 |
| **03** | 找位置 | 做多时用「前低」作支撑、均线作趋势参考 |
| **04** | 看信号 | 止跌：旭日东升（阳包阴）或 RSI 超卖后回升 |
| **05** | 定止损 | 左侧：前低；右侧：信号 K 线低点；兜底：ATR 倍数；总风险 = 本金 × 1.5% |
| **06** | 定仓位 | 以损定量：仓位 = 总风险 / (入场价 − 止损价) |
| **07** | 看空间 | 前高或目标位至少满足 2:1 盈亏比，否则不开仓 |
| **08** | 开仓 | 满足 01–07 后，按计算出的仓位下单 |
| **09** | 平仓 | 打止损必走；盈亏比达 2:1 减半仓；剩余仓位跟踪止盈（ATR 上移） |
| **10** | 加仓 | 仅在有浮盈且再次出现买点信号时加仓，最多加 1 次，按新开仓方式以损定量 |

## 可调参数（在 `initialize` 中）

| 参数 | 默认 | 说明 |
|------|------|------|
| `g.security` | 510300.XSHG | 交易标的 |
| `g.short_ma` / `g.long_ma` | 10 / 30 | 辨趋势均线周期 |
| `g.trend_threshold` | 0.02 | 均线偏离在此内视为震荡 |
| `g.rsi_period` / `g.rsi_oversold` | 14 / 30 | RSI 周期与超卖线 |
| `g.risk_pct` | 0.015 | 总风险 = 本金 × 1.5% |
| `g.stop_atr_mult` | 1.5 | 止损兜底：入场价 − ATR×倍数 |
| `g.min_reward_ratio` | 2.0 | 最低盈亏比 2:1 |
| `g.add_only_if_profit` | True | 仅浮盈时加仓 |
| `g.max_add_times` | 1 | 最多加仓次数 |

## 如何运行与回测

1. 登录 [聚宽](https://www.joinquant.com) → 进入「回测」。
2. 新建回测，将 `complete_flow_system.py` 的 **完整代码** 复制到编辑器。
3. 设置回测区间（建议 2 年以上）、初始资金、频率选「日」。
4. 运行回测，观察收益曲线、最大回撤、夏普、胜率、交易次数等。

## 关于盈利

- 策略按流程图实现逻辑闭环，**不保证盈利**；能否盈利取决于标的、区间、参数与市场环境。
- 建议多区间、多标的回测，并做参数敏感性检查，再考虑实盘或模拟盘小资金验证。


 设计 Part 1：市场环境分档（大盘过滤）

  新增 get_market_regime(context) 函数，以沪深300（000300.XSHG）的 MA60 为基准：

  ┌──────┬──────────────────────────────────┬───────────────┬──────────────┐
  │ 档位 │             判断条件             │ max_positions │ 仓位上限系数 │
  ├──────┼──────────────────────────────────┼───────────────┼──────────────┤
  │ 牛市 │ 沪深300 收盘 > MA60 且偏差 > +2% │ 2             │ 100%         │
  ├──────┼──────────────────────────────────┼───────────────┼──────────────┤
  │ 震荡 │ 沪深300 收盘 在 MA60 ±2% 以内    │ 1             │ 70%          │
  ├──────┼──────────────────────────────────┼───────────────┼──────────────┤
  │ 熊市 │ 沪深300 收盘 < MA60 且偏差 < -2% │ 1             │ 40%          │
  └──────┴──────────────────────────────────┴───────────────┴──────────────┘

  关键变化：
  - 原来 g.max_positions = 2 是固定的，改为每周由 get_market_regime 动态返回
  - 仓位上限系数作用于 calc_trade_value 中的 slot_cap 计算
  - 大盘数据通过 attribute_history('000300.XSHG', ...) 获取，与个标ETF数据获取方式一致


设计2： 
新增 calc_momentum_score(prices) 函数，对每个ETF计算 0～100 的综合评分，取代原来的 get_signal() 强/弱二元判断：

  评分由三个子项构成（各满分约33分）：

  ┌──────────┬────────────────────────────────────────────────────────────────┬──────┐
  │   子项   │                            计算方式                            │ 满分 │
  ├──────────┼────────────────────────────────────────────────────────────────┼──────┤
  │ 突破强度 │ (当前价 - 20日最高) / 20日最高，仅当突破时有分，最大贡献上限5% │ 40分 │
  ├──────────┼────────────────────────────────────────────────────────────────┼──────┤
  │ RSI强度  │ RSI在45～70区间内线性映射：45→0分，70→30分                     │ 30分 │
  ├──────────┼────────────────────────────────────────────────────────────────┼──────┤
  │ 均线斜率 │ (短均线 - 长均线) / 长均线，范围0～3%线性映射到0～30分         │ 30分 │
  └──────────┴────────────────────────────────────────────────────────────────┴──────┘

  入场门槛： 综合得分 ≥ 30分才进入候选（替代原来的趋势threshold + 信号判断双重过滤）

  趋势判断调整：
  - 原 get_trend() 的 trend_threshold=0.01 仍保留，但仅作为「排除明确空头」的粗过滤（trend != 'down' 即可，不再要求必须 'up'）
  - 震荡期（flat）的ETF只要评分达标，也可进入候选
  - 每周按评分降序排列候选，取前 max_positions 名开仓

  优先级变化：
  - 原来：强信号(rank=0) > 弱信号(rank=1)，同级按ETF池顺序
  - 现在：统一按动量评分降序，分数高者优先，逻辑更连续


    止损微调：
  - trend_threshold 从 0.01 降至 0.003（0.3%），让震荡期ETF也能通过趋势粗过滤
  - fixed_stop_pct 从 0.08 维持不变（8%止损与20%最大回撤目标匹配）
  - profit_trigger 从 0.05 降至 0.03（浮盈3%即启动跟踪止盈，更早锁定利润）

  不变的部分：
  - 周度调仓频率不变
  - 止损计算逻辑（calc_stop）不变
  - 加仓逻辑（Step 10）不变
  - ETF池不变
