# 完整交易系统流程说明

本说明对应策略文件：`strategies/complete_flow_system.py`，基于「最完善的交易流程图」实现可运行、可回测的 10 步流程。

---

## 流程与代码对应关系

| 步骤 | 流程图名称 | 实现要点 |
|------|------------|----------|
| **01** | 辨趋势 | 双均线（短/长）：短 > 长 → 上涨；短 < 长 → 下跌；均线缠绕 → 震荡 |
| **02** | 判方向 | 趋势为上涨 → 主多；下跌 → 主空（本策略仅做多）；震荡 → 偏多 |
| **03** | 找位置 | 做多时用「前低」作支撑、均线作趋势参考 |
| **04** | 看信号 | 止跌：旭日东升（阳包阴）或 RSI 超卖后回升 |
| **05** | 定止损 | 左侧：前低；右侧：信号 K 线低点；兜底：ATR 倍数；总风险 = 本金 × 1.5% |
| **06** | 定仓位 | 以损定量：仓位 = 总风险 / (入场价 − 止损价) |
| **07** | 看空间 | 前高或目标位至少满足 2:1 盈亏比，否则不开仓 |
| **08** | 开仓 | 满足 01–07 后，按计算出的仓位下单 |
| **09** | 平仓 | 打止损必走；盈亏比达 2:1 减半仓；剩余仓位跟踪止盈（ATR 上移） |
| **10** | 加仓 | 仅在有浮盈且再次出现买点信号时加仓，最多加 1 次，按新开仓方式以损定量 |

---

## 可调参数（在 `initialize` 中）

| 参数 | 默认 | 说明 |
|------|------|------|
| `g.security` | 510300.XSHG | 交易标的 |
| `g.short_ma` / `g.long_ma` | 10 / 30 | 辨趋势均线周期 |
| `g.trend_threshold` | 0.02 | 均线偏离在此内视为震荡 |
| `g.rsi_period` / `g.rsi_oversold` | 14 / 30 | RSI 周期与超卖线 |
| `g.risk_pct` | 0.015 | 总风险 = 本金 × 1.5% |
| `g.stop_atr_mult` | 1.5 | 止损兜底：入场价 − ATR×倍数 |
| `g.min_reward_ratio` | 2.0 | 最低盈亏比 2:1 |
| `g.add_only_if_profit` | True | 仅浮盈时加仓 |
| `g.max_add_times` | 1 | 最多加仓次数 |

## 如何运行与回测

1. 登录 [聚宽](https://www.joinquant.com) → 进入「回测」。
2. 新建回测，将 `complete_flow_system.py` 的 **完整代码** 复制到编辑器。
3. 设置回测区间（建议 2 年以上）、初始资金、频率选「日」。
4. 运行回测，观察收益曲线、最大回撤、夏普、胜率、交易次数等。

---

## 关于盈利

- 策略按流程图实现逻辑闭环，**不保证盈利**；能否盈利取决于标的、区间、参数与市场环境。
- 建议多区间、多标的回测，并做参数敏感性检查，再考虑实盘或模拟盘小资金验证。
