# 策略能否盈利 & 如何测试/回测

## 一、这些策略能盈利吗？

**没有保证。** 能否盈利取决于：

| 因素 | 说明 |
|------|------|
| **市场环境** | 趋势市适合均线/突破/动量；震荡市适合均值回归/网格/布林带；不同阶段表现会变 |
| **标的与周期** | 同一策略在不同标的（指数ETF、个股）和不同回测区间下结果差异很大 |
| **参数** | 均线周期、RSI 阈值、网格档数等都会影响收益与回撤，需要调参或稳健性检验 |
| **成本与滑点** | 已设手续费，实盘还有滑点、冲击成本，会略差于回测 |

**回测的作用**：用历史数据看策略在“过去”的表现（收益、回撤、夏普等），**不能预测未来**。回测好 ≠ 未来一定盈利，但回测差通常要慎用。

**建议**：
- 多测几个回测区间（含牛市、熊市、震荡）。
- 对比基准（如沪深300），看是否跑赢、回撤是否可接受。
- 调参时避免过度拟合（参数不要为某一段行情“量身定制”）。

## 二、在聚宽上如何回测（推荐）

本仓库的策略是按 **聚宽 (JoinQuant)** 的 API 写的，**在聚宽网站做回测最省事**。

### 步骤 1：打开回测

1. 打开 [https://www.joinquant.com](https://www.joinquant.com) 并登录。
2. 顶部或左侧进入 **「回测」**（或「研究」里新建「回测」）。

### 步骤 2：粘贴策略代码

1. 新建回测，在代码编辑区 **删掉默认代码**。
2. 打开本仓库里对应策略的 `.py` 文件（如 `strategies/ma_crossover.py`），**整份复制** 到聚宽编辑器。

### 步骤 3：设置回测参数

在回测页面的参数区（一般在右侧或下方）设置：

| 参数 | 建议值 | 说明 |
|------|--------|------|
| **回测开始时间** | 如 2020-01-01 | 至少包含一轮牛熊或 2–3 年，更有参考意义 |
| **回测结束时间** | 如 2024-12-31 | |
| **初始资金** | 如 1000000（100 万） | 按你关心的资金量设 |
| **频率** | **日** | 策略按日调仓，选「日」即可 |

### 步骤 4：运行并看结果

1. 点击 **「运行回测」**（或「编译并运行」）。
2. 跑完后看：
   - **收益曲线**：策略 vs 基准（如沪深300）。
   - **年度收益**、**最大回撤**、**夏普比率**、**胜率**、**交易次数** 等。

### 步骤 5：多策略、多区间对比

- 换不同策略文件重复 2–4 步，对比同一区间下的表现。
- 同一策略再换 2–3 个时间段（如 2018–2020、2021–2023）各跑一遍，看是否稳定。

## 三、回测时重点看哪些指标

| 指标 | 含义 | 粗略参考（非标准） |
|------|------|---------------------|
| **年化收益** | 折算成一年的收益率 | 与基准比，是否跑赢、幅度如何 |
| **最大回撤** | 从高点到低点的最大跌幅 | 能否接受（如 <20%、<30%） |
| **夏普比率** | 单位风险的超额收益 | >1 较好，>2 很优（和频率、无风险利率有关） |
| **胜率** | 盈利次数 / 总交易次数 | 结合盈亏比一起看，不是越高越好 |
| **交易次数** | 总买卖次数 | 太少可能样本不足，太多要关注成本 |

**注意**：回测默认是“理想成交”（如开盘价成交），实盘会有滑点与冲击，结果通常会略差于回测。

## 四、本地能否回测？

- 策略代码依赖聚宽的 **回测引擎 API**（如 `initialize`、`run_daily`、`attribute_history`、`order_target` 等），**不能直接在本地用普通 Python 跑**。
- 若要在本地跑，需要：
  - 使用聚宽提供的 **本地研究/回测环境**（若有），或  
  - 用 **joinquant 的本地/离线回测 SDK**（若官方有提供），按官方文档配置数据与运行方式。

当前最省事的做法仍是：**在聚宽网站复制策略 → 设置时间与资金 → 运行回测**，用上面的指标做对比和评估。

---

## 五、简要结论

| 问题 | 答案 |
|------|------|
| 这些策略能盈利吗？ | 不保证。取决于市场、标的、参数；需通过回测看历史表现。 |
| 怎么测试？ | 在聚宽回测里粘贴策略代码，设好时间、资金、频率，运行回测。 |
| 怎么判断好不好？ | 看年化收益、最大回撤、夏普、胜率、交易次数，并做多区间、多策略对比。 |

把「能否盈利」交给回测结果和实盘小资金验证，而不是单凭策略类型下结论。
